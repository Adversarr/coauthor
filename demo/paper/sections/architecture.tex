\subsection{Overview}
\label{subsec:overview}

Seed uses a layered architecture that separates domain decisions, application logic, infrastructure adapters, and user interfaces. This structure supports testability and controlled evolution of each layer.

\subsection{Core Components}
\label{subsec:components}

\begin{itemize}
    \item \textbf{Core Layer}: domain entities, event contracts, and service ports.
    \item \textbf{Application Layer}: task lifecycle, interaction handling, projections, and audit queries.
    \item \textbf{Infrastructure Layer}: JSONL persistence, tool execution, LLM clients, HTTP/WS transport.
    \item \textbf{Interface Layer}: terminal UI and web UI built on shared backend contracts.
\end{itemize}

\subsection{Execution Planes}
\label{subsec:event-flow}

\begin{enumerate}
    \item \textbf{Domain event plane}: append-only lifecycle and interaction events.
    \item \textbf{Audit plane}: tool request/result records with execution metadata.
    \item \textbf{UI plane}: realtime updates for streaming output and interaction state.
\end{enumerate}
